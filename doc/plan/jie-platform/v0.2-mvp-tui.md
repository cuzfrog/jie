# jie-platform 0.2 — TUI MVP

The v0.2 deliverable: a working TUI in `packages/jie-tui/` that replaces the v1 stub (`throw "TUI not implemented..."`). Drives the full team interactively.

**Specs (read in this order):**

- `ui/tui-overview.md` — entry function, bus contract, role, invariants, multi-team, flag parity, test strategy. Children: `ui/tui-state.md` (data model, reducer rules), `ui/tui-layout.md` (spatial design), `ui/tui-shortcuts.md` (keybindings, slash commands), `ui/tui-pi-reference.md` (library, theme tokens), `ui/tui-user-scenarios.md` (T1–T6).
- `ui/tui-state.md` — `TuiState` shape, `Block`/`Card`/`Turn` types, reducer rules per topic, per-agent streaming isolation, queue-pickup debounce, transient messages, editor → focused agent wiring.
- `ui/tui-layout.md` — spatial design: bottom strip (body, editor, footer), chat pane order (user `›`, assistant `●`, thinking `Thinking...`, tool cards, working indicator), rail width formula, editor with history, footer line 1 (CWD + branch + team:agent) and line 2 (stats + hint + model), borders/separators.
- `ui/tui-shortcuts.md` — keybinding matrix and semantics (`Ctrl+C` clear/quit, `Ctrl+D×2` quit, `Esc×2` interrupt, `Ctrl+T`/`Ctrl+O` toggles, prompt history, rail cycling).
- `ui/tui-pi-reference.md` — pi theme tokens (256-color, truecolor, hex), font treatments, separator conventions, footer styling, markdown component behavior.
- `ui/cli.md` — `jie` (TUI), `jie -p`, `--team`, `--api-key`, `--resume` / `--continue` flags, slash-command counterparts.

**AC:** `00-tui-user-scenarios.md` (T1–T5).

## Scope

**In scope (v0.2):**

- All T1–T5 scenarios from `00-tui-user-scenarios.md`.
- Multi-team in the TUI (`loadTeam`, `/team` slash command, swap-back event buffer) — per `ui/tui-overview.md` "Boundary with the platform".
- Full flag parity with `jie -p` (`--team`, `--api-key`, `--resume`, `--continue`) — per `ui/tui-overview.md` "Flag parity".
- Library: `@earendil-works/pi-tui` (sister package to `pi-ai` / `pi-agent-core`; both already vendored).
- Hand-recorded JSONL fixtures under `tests/e2e/tui/fixtures/`; replay-based TUI tests under `tests/e2e/tui/`.

**Out of scope (v0.2 stays Day 2+):**

- The jie-team dev team (`specs/jie-team/`). v0.2 TUI runs against any user-installed team, but the dev team itself is not in v0.2.
- MCP client (`specs/jie-platform/10-configuration.md` "MCP Server Configuration"). Archived scenario 7 stays archived. The CLI's "Behavior" step 6 (MCP connection) is a no-op in v0.2.
- jie npm publish / install script / `jie upgrade` (backlog #20).
- TUI a11y / theme customization.
- Cross-process team visibility.
- vim-style keybindings (Day 2+ add-on).
- Hand-rolled raw-mode renderer / Ink / blessed (decision: pi-tui; the TUI does not own the rendering layer).
- LLM-driven recording of event traces (Day 2+; v0.2 fixtures are hand-recorded).

**v0.2 CLI surface change:** `jie` (no flags) launches the TUI. Previously this was an error / no-op; v0.2 makes the TUI the default surface. The `-p` mode keeps its single-team, one-shot shape; it does not adopt `loadTeam`. See `ui/cli.md` "Behavior" step 8.

## Phases

### Phase 1: render primitives + input loop (T1 smoke)

- Replace the v1 stub with a real `createTui` that:
  - Checks `process.stdin.isTTY`; logs the non-TTY error and returns non-zero otherwise.
  - Constructs `ProcessTerminal` and `TUI` from `pi-tui`.
  - Mounts a `Container` with `AgentsRail`, `ChatPane`, `Editor`, `Footer` — minimal, hard-coded content ("hello agent").
  - Listens for `Ctrl+D`; restores TTY; exits 0.
  - Deletes the v1 `throw "TUI not implemented"` from `packages/jie-tui/index.ts`; the real `createTui` replaces it.
  - Wires the `roles: string[]` plumbing: until the first `team.{teamId}.loaded` arrives, the TUI renders a placeholder rail from `roles` (role names only) and a `Loader` in the chat pane area (per `ui/tui-state.md` "Roles bootstrap").
- Spec: `ui/tui-overview.md` "Initial bootstrap", `ui/tui-shortcuts.md` "Ctrl+D×2 to quit".
- AC: `jie` opens a TUI; the layout matches the figure in `ui/tui-layout.md` "Bottom strip"; `Ctrl+D` exits 0.
- Test: `tests/e2e/tui/boot.test.ts` drives a fake `VirtualTerminal`, asserts the boot frame and the exit on `Ctrl+D`.

### Phase 2: state derivation (reducer)

- Define `TuiState` in `packages/jie-tui/state.ts` per `ui/tui-state.md` "State shape".
- Define `reduce(state: TuiState, envelope: EventEnvelope): TuiState` — pure per the topic rules in `ui/tui-state.md` "Reducer rules":
  - `team.{id}.loaded` → seed `agents` from `payload.agents` (with team-switch reset).
  - `team.{teamId}.agent.{agentKey}.prompt` → open a fresh `currentTurn`; move the prior populated one to history.
  - `agent.turn.start` → set agent status `busy`; open / advance turn; clear `errorBanner` (the Editor's `onSubmit` / `onCancel` handlers also publish `ui.error.clear` for the no-model-selected path).
  - `agent.idle` → set agent status `idle`; record `lastIdleAt` for the queue-pickup flicker debounce.
  - `agent.stream.chunk` → append to the matching `Block` in `currentTurn.blocks`.
  - `agent.stream.end` → no-op.
  - `agent.tool.call` / `agent.tool.result` → push / match the `Card` on `currentTurn.cards`.
  - `agent.prompt.queue.update` → replace `queue`.
  - `ui.*` topics (`rail.toggle`, `agent.cycle`, `thinking.toggle`, `tool.toggle`) → local UI mutations.
- AC: `reduce` is a referentially-transparent function. Test it against the T1 trace.
- Spec: `ui/tui-state.md` "State shape", "Initial state", "Reducer rules", "Per-agent streaming isolation".

### Phase 3: layout (AgentsRail, ChatPane, Footer)

- Implement each pane as a `Component` subclass with `render(width: number): string[]`.
- Bottom strip anatomy per `ui/tui-layout.md` "Bottom strip": body (chat, optionally with rail) → editor → 2-line footer. Editor and footer are full-width; the rail sits inside the body when visible.
- Chat pane order per `ui/tui-layout.md` "Chat pane": user prompt `›` (cyan), assistant text `●` (accent), thinking block `Thinking...` italic in `thinkingText`, tool cards, working indicator. One blank line between turns.
- Footer line 1 (identity strip) per `ui/tui-layout.md` "Footer": `cwd (branch)` left in `accent`, `teamId:focusedAgentKey` right in `muted`. Footer line 2 (state + hint + model): stats left, rail-state-dependent hint, `(provider) modelId | effort` right with the model id in `accent`.
- Per-agent streaming isolation: switching focus does not cancel another agent's stream. The reducer's per-agent map (`state.agents: Map<agentKey, AgentUiState>`) is the structural enforcement; the renderer reads only the focused agent's history + current turn.
- Spec: `ui/tui-layout.md` (whole file), `ui/tui-state.md` "Editor → focused agent".
- AC: T1's first frame matches the figure in `ui/tui-layout.md` "Bottom strip". Snapshot file: `tests/e2e/tui/__snapshots__/t1-boot.snap`.

### Phase 4: slash commands

- `/login` — open a `SelectList` of providers from `pi-ai`'s `getProviders()`; on select, open a hidden `Input` for the API key; write to `~/.jie/auth.json` (mode `0600`). Transient message: `logged in to <provider>`.
- `/logout [<id>]` — clear one or all entries.
- `/model <p>/<m>` — validate and write to `~/.jie/settings.json`. Transient: `default model set to <p>/<m>`.
- `/team [<id>]` — open the team picker (`SelectList` with `fuzzyFilter` over `teamRegistry.listInstalled()`); on select, call `handle.loadTeam(id)`. With a positional `<id>`, validate and call `loadTeam` directly. Transient on success: `default team set` (only on direct write — picker selection does not write settings, just switches view).
- `/team --unset` — clear `defaultTeam`. Mid-session unset is **not** supported.
- `/team not-installed` — render the error `team '<id>' is not installed; checked .jie/teams/<id>/ and ~/.jie/teams/<id>/` in the input area. No write. No team switch.
- `/clear` — clear `state.agents[*].history` and `state.agents[*].currentTurn`. Memory rows on disk are untouched.
- `/help` — open an overlay rendering the keymap from `tui-shortcuts.md`.
- `/exit` — same as `Ctrl+D`.
- Transient messages: reducer tracks `state.transientMessage: { text, shownAt } | null`; renderer hides it after 5 s or on next `Enter`, whichever comes first (`ui/tui-state.md` "Transient messages").
- Spec: `ui/tui-shortcuts.md` "Slash commands", `ui/tui-state.md` "Actions" (transient / clearTransient).
- AC: T4's first-time setup flow works end-to-end.

### Phase 5: streaming + tool cards + thinking

- Append `agent.stream.chunk` content to the matching `Block` in `state.agents[focused].currentTurn.blocks` (no per-chunk re-render of the chat history above). Reducer is append-only; the block carries its final length when the next chunk of a different `block_type` arrives.
- Tool cards: render the call as `● <name>  <input>` on `agent.tool.call`; render the result as `✓ <name>  <ms>ms` (or `✗ <name>  <error>`) on `agent.tool.result`. Track `card.expanded` per card; `Ctrl+O` toggles all (not per-card).
- Thinking blocks: render `Thinking...` italic in `thinkingText` when collapsed; render the body's `Markdown(text, 1, 0, mdTheme, { color: thinkingText, italic: true })` when expanded. Default collapsed. `Ctrl+T` toggles all.
- Working indicator (`● Working…`): shown whenever the focused agent's status is `busy` (any phase — pre-stream, thinking, streaming, awaiting tool result, post-tool). Spinner frame advances with `Date.now()` so the animation continues even when the focused agent changes.
- Queue-pickup flicker: reducer sets `lastIdleAt` on `agent.idle`; renderer forces a `●busy` display when `Date.now() - lastIdleAt < 50`. Render-side only — the reducer stays pure.
- Spec: `ui/tui-state.md` "Reducer rules" (`agent.stream.chunk`, `agent.tool.call`, `agent.tool.result`), "Queue-pickup flicker"; `ui/tui-layout.md` "Chat pane" (item 3 thinking, 4 tool cards, 5 working indicator); `ui/tui-shortcuts.md` "Ctrl+T semantics", "Ctrl+O semantics".
- AC: T2's `manager → worker` flow renders the tool cards inline; T5's queued-prompt indicator renders the queue depth.

## Functional AC

`doc/specs/jie-platform/ui/tui-user-scenarios.md` (T1–T6). Plus the cross-cutting AC from `ui/tui-overview.md`:

- **Non-TTY refuses.** `jie` (no flags) under a piped stdout logs the non-TTY error and exits non-zero.
- **Ctrl+C interrupts a turn.** Mid-turn `Esc Esc` (or `Ctrl+C`) publishes a synthetic interrupt; the in-flight LLM call's `AbortSignal` fires; the TUI does not exit. See `ui/tui-shortcuts.md` "Esc×2 vs Ctrl+C".
- **Ctrl+D exits cleanly.** With a turn in flight, prompts for confirmation; without one, exits 0 directly. See `ui/tui-shortcuts.md` "Ctrl+D×2 to quit".
- **Working indicator persists across agent switch.** Per `ui/tui-layout.md` "Chat pane" item 5 and `ui/tui-state.md` "Per-agent streaming isolation"; the spinner frame advances with `Date.now()`, so cycling focus while another agent is busy does not freeze the indicator.
- **No queue-pickup flicker.** When the body picks up the next queued prompt, the brief `agent.idle` → `agent.turn.start` window is hidden by the 50 ms `lastIdleAt` debounce. See `ui/tui-state.md` "Queue-pickup flicker".
- **Footer line 1 is identity, line 2 is state.** Per `ui/tui-layout.md` "Footer" — no shortcuts on line 1, `cwd (branch)` left and `teamId:focusedAgentKey` right.
- **Snapshot tests run in CI.** All five T1–T5 JSONL traces replay; the rendered frames match the snapshots.

## E2E test backend

Same as v1: the TUI scenarios (T1–T5) replay recorded traces, not live LLM calls. The CI backend (NVIDIA Nemotron) is used by the v1 `tests/e2e/v1-scenarios.test.ts` (not the TUI tests). The TUI tests are deterministic by construction.
