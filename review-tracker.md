### Batch C — memory model
7. ~~**`compact()` vs `persist()` for `compactionSummary`**~~ — closed 2026-06-13. Resolved by Items 7, 8, 9 together (see below). The two writers are now structurally separate: `persist()` is unconditional in the body's `message_end` listener; `compact()` is the sole writer for `compactionSummary` rows and is called from the body's `transformContext` wrapper, not from any pi-agent event. Role union cleaned up: `bashExecution` and `custom` removed (Jie's `bash` returns `toolResult`; no custom messages in v1). Edits: `08-memory.md` "Compact" + "Integration with pi-agent" + TurnRecord role row + role source row, `06-agent-model.md` AgentOptions `transformContext` row + event bridging `message_end` and `agent_end` rows + Memory Persistence section.
8. ~~**`subscriberCount` vs self-receipt**~~ — closed 2026-06-13. Subtract self from the LLM-facing count, conditional on the publisher being subscribed to the topic (`AgentSoul.subscriptions.includes(topic)`). `details.recipients` carries the same LLM-facing number. Bus-level `subscriberCount` is unchanged. Edits: `06-agent-model.md` "Built-in Tool: `notify`" step 3 + step 4; ADR 9 §4 wording clarified (no new ADR).
9. ~~**`compact()` transactionality**~~ — closed 2026-06-13. Resolved together with item 7: `compact()` writes the summary row + the raw range's `compacted=1` flags inside a single `storage.transaction()`. Crash mid-`compact()` rolls back the transaction; the body's `transformContext` wrapper detects the summary on the next turn and re-runs `compact()` (idempotent). Edits: `08-memory.md` "Compact" section. ADR: none for items 7+9 together — the redesign is a spec correction (writer split + transaction wrap), not an architectural shift; the new design folds cleanly into the existing Memory layer.
